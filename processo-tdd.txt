O processo apresentado na figura evidencia o que é feito quando se escreve os testes funcionais primeiro e, se ele não passa, então se escreve teste unitário até que ele passe, sempre desenvolvendo o código de produção depois que se testa, até que chegue o momento para um possível refactor. Assim, após esse ciclo, se rodam os testes funcionais de novo e a cada falha desse fluxo, se rodam testes unitários novamente. Dessa forma, para cada teste funcional que falha se passa pelo o ciclo de testes unitários novamente.
